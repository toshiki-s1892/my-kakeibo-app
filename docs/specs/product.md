# 要件定義

プロダクト全体の背景・対象ユーザー・機能要件・非機能要件をまとめたもの。画面ごとの詳細な仕様（バリデーション・業務フロー・API）は[overview.md](./overview.md)・[features/](./features/)を参照。

## プロダクト概要

**アプリ名: 「かけぼ」**

個人・家族向けの家計簿 Web アプリケーション。収入・支出の記録・管理を行い、**AI によるレシート読み取り・支出分析・アドバイス**をコア価値として提供する。

## ターゲットユーザー

- 家計管理をしたい個人・家族
- AI を活用して手間なく家計を把握・改善したいユーザー

## 機能要件

### 認証

- Clerk を用いたサインアップ・サインイン
- サインイン済みユーザーのみがアプリを利用できる

### プロフィール設定

- 初回ログイン時にユーザー情報を登録する
  - 居住都道府県（1〜47）
- プロフィール登録時に「本人」レコードを `family_members` に自動作成する（relationship_code: 1）
  - 表示名（最大 50 文字）
  - 性別（0: 不明 / 1: 男性 / 2: 女性 / 9: 適用不能）
  - 誕生日
- 家族構成の登録（任意・複数可）
  - 表示名（最大 50 文字）
  - 続柄（1: 本人 / 2: 配偶者 / 3: 子 / 4: 親 / 5: その他）
  - 性別
  - 生年

### カテゴリ管理

- デフォルトカテゴリをあらかじめ用意する（食費・交通費・光熱費 など）。`categories.user_id` が NULL のレコードをシステム共通カテゴリとして扱う
- ユーザーは独自カテゴリの追加・編集が可能
- カテゴリにはタイプ（支出 / 収入）を設定する
- カテゴリ名は最大 50 文字

### 取引記録

- 収入・支出の手動入力
  - 家族メンバー（誰の取引か。必須。本人を含む `family_members` から選択）
  - 金額
  - 取引日
  - カテゴリ
  - 取引先（任意。店舗・取引相手を記録する補助的なマスタ。詳細は [features/transaction-parties.md](./features/transaction-parties.md)）
  - 詳細（任意・最大 255 文字。旧称「メモ」）
- AI レシート読み取りによる自動入力（→ AI 機能）

### 定期取引

- 給料・家賃・保険料など、毎月/毎年決まったタイミングで発生する収入・支出をテンプレートとして登録できる
- テンプレートに基づき、Vercel Cron Jobsで自動的に取引記録を生成する（詳細: [recurring-transactions.md](../specs/features/recurring-transactions.md)）
- 頻度は毎月・毎年の2種類。終了日は任意（未設定なら無期限）

### 集計・レポート

- 月次サマリー（収入・支出合計・残高）
- カテゴリ別グラフ（円グラフ・棒グラフ）
- 期間フィルター（日付範囲指定）
- 家族メンバー別集計

### AI 機能

利用AIサービス: **Google Gemini API**（画像認識・テキスト生成の両方に対応）

| featureCode | 機能 | 概要 |
|---|---|---|
| 1 | レシート読み取り・自動入力 | レシートの画像から金額・店舗・日付を読み取り、取引として自動登録する |
| 2 | 支出分析・アドバイス | 月の支出パターンを分析し、節約提案や異常検知を行う |

- AI 機能の利用は `ai_usage_logs` テーブルに記録する
- Gemini API はサーバーサイド（Honoハンドラ内）でのみ呼び出す

## 非機能要件

| 項目 | 要件 |
|---|---|
| 認証 | Clerk による外部認証で安全性を担保 |
| データ管理 | ソフトデリート対応（users テーブル: deletedAt） |
| スケーラビリティ | Turso（分散 SQLite）により将来的な拡張に対応 |
| 型安全性 | Zod によるスキーマバリデーション |
| AI 利用管理 | AI 機能の利用ログを記録し、乱用・コスト管理に活用 |
