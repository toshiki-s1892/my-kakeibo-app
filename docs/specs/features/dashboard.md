# ダッシュボード

## 概要

世帯全体の合計という概念を持たず、**家族メンバーごとのダッシュボードをスライドで切り替えて見る**構成にする。「合計を見ても誰が使ったか分からない」ため、最初から個人単位の情報を主役にする。

```
┌─────────────────────────────────┐
│  « 2026年6月 »      ● ○ ○        │  ← ドットインジケーター
├─────────────────────────────────┤
│         ← 本人 のダッシュボード →   │
│  支出 ¥90,000  収入 ¥300,000     │
│  収支 +¥210,000                  │
│  💬 AI簡易分析（本人向け）          │
│  カテゴリ別支出（円グラフ）         │
│  月ごとの収支推移（過去6カ月）      │
└─────────────────────────────────┘
       （左右スワイプで配偶者・子へ切り替え）
```

## 月切り替え

`«»`で前月・翌月に1ヶ月単位で移動する（任意の日付範囲指定は行わない）。月の切り替えは全スライド共通（どのメンバーを見ていても同じ月を参照する）。

## メンバースライド

- 家族メンバーごとに1枚のスライド（カード）を持つ。スライド内の全データ（サマリー・AI簡易分析・グラフ）はそのメンバーの`family_member_id`でフィルタされた取引データのみを参照する
- 表示順は**本人（`relationship_code = SELF`）を先頭、以降は作成順**（論理削除済みメンバーはスライドから除外）
- 世帯全体を合計した値・グラフは表示しない（個人ごとの値のみ）
- 切り替え操作は**矢印（←/→）+ドットインジケーターの併用**とする。矢印でPC・キーボード操作に対応し、ドットでメンバー数・現在位置を把握、タッチ操作ではスワイプも可能にする

## スライド内の構成

### 1. サマリー

選択中の月・選択中のメンバーの支出合計・収入合計・**収支**（収入−支出。銀行口座連携はないため、この期間の収支差額を指す。「残高」という名称は累積資産を想起させるため使わない）。

### 2. AI簡易分析

[AI機能の簡易支出分析](./ai.md#2-簡易支出分析expense_analysis)。メンバーごとに1日1回生成・キャッシュする（スライドを切り替えるたびにGeminiを呼ぶとコストが大きいため）。

### 3. カテゴリ別グラフ（円グラフ）

選択中の月・選択中のメンバーの**支出のみ**をカテゴリ別に集計した円グラフ。件数に依存しない動的カラー生成を行う（カテゴリ数に上限を設けない方針との整合性。[categories.md](./categories.md#カテゴリ数の上限について)参照）。

**表示カテゴリの決め方**: 単純な金額上位N件だと、カテゴリの分け方が世帯ごとに異なる（例: 「光熱費」を1つにまとめるか「電気代」「ガス代」「水道代」に分けるか）ため、分け方が細かい世帯ほど重要なカテゴリが上位から漏れやすく、また月ごとに表示されるカテゴリが入れ替わって比較しづらい。そのため以下の手順で表示するカテゴリを決める。

1. **親カテゴリへの合算**: `parentId`を持つカテゴリの金額は親カテゴリに合算してから以降の判定を行う（[categories.md](./categories.md#親カテゴリグラフ上のグルーピング)参照）。合算後のグループを以下では単に「カテゴリ」と呼ぶ
2. **ピン留め**（`categories.isPinned = true`のカテゴリ、または子のいずれかがピン留めされているグループ）のうち、**選択中の月・メンバーでの金額が0より大きいもの**を優先的に最大5枠まで表示（[categories.md](./categories.md#カテゴリの固定表示ピン留め)参照）。金額が0のピン留めカテゴリは枠を消費しない（例: 子供は住居費・光熱費を直接使わないことが多いため、本人向けではピン留め表示されても子供向けでは表示されない）
3. 残りの枠を金額が多い順に埋める
4. 枠に入らなかったカテゴリは「その他」に集約

ピン留めされたカテゴリ（グループ）のうち金額が0より大きいものの数が5件を超える場合は、**5件という上限より優先してそれらを全て表示する**（ユーザーが明示的に固定表示を選んだものを自動で「その他」に隠すと混乱を招くため）。この場合、金額上位による自動補完は行わない。

選択中の月・メンバーに**支出が1件もない場合**は、空状態（例:「この月の支出はまだありません」）を表示する。

### 4. 月ごとの収支推移（棒グラフ）

選択中のメンバーの**過去6ヶ月分**（選択中の月を含む）の収支（収入−支出）を並べた棒グラフ。月切り替えと連動し、選択中の月を起点に過去6ヶ月分を表示する。

## バックエンド

- 取引データの集計は`server/lib/`にヘルパー関数として切り出し、月・`familyMemberId`でフィルタする（CLAUDE.mdのDALパターンに従う）
- カテゴリ別集計・月次推移はDBクエリで集計してから返す（全件取得してクライアント側で集計しない）
