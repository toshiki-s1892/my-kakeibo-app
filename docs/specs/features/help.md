# ヘルプ

`(app)/help`に配置する、操作に迷いやすい機能を補足説明するページ。`(auth)`・`(onboarding)`からはアクセスしない（[アプリのルートグループ構成](./profile-setup.md#ルートグループ構成)参照、ログイン済みかつセットアップ完了済みのユーザーのみが対象）。

## 背景

[カテゴリ管理](./categories.md)の親カテゴリ機能・ピン留め機能は、その効果（円グラフでの表示への影響）が画面上の操作だけでは伝わりにくい。これらの機能はカテゴリ作成・編集モーダルや一覧画面に常設のヘルプテキストを置く方針だが、モーダル内は表示領域が限られるため図解までは置けない。そこで、図解を含む詳しい説明はヘルプページに集約し、モーダル・一覧画面からはそこへのリンクで導線をつなぐ。

## 入口

ナビゲーション項目・専用のヘルプアイコンは追加しない（[architecture/overview.mdの`(app)`レイアウト構成](../../architecture/overview.md#appレイアウト構成)が「フッターなし・ナビ5項目固定・専用設定項目を追加しない」というミニマム方針を取っているため、これと一貫させる）。

- **どこからでも開ける入口**: ヘッダー右上のClerkの`UserButton`のドロップダウンに、`<UserButton.Link label="ヘルプ" href="/help" />`（[Clerk公式: Add custom items and links to the UserButton component](https://clerk.com/docs/nextjs/guides/customizing-clerk/adding-items/user-button)）でカスタム項目を追加する。既存の「Manage account」「Sign out」と並ぶ形で表示される
- **文脈に応じた直接リンク**: [カテゴリ作成・編集モーダルの親カテゴリ項目の常設ヘルプテキスト](./categories.md#親カテゴリグラフ上のグルーピング)から`/help#categories`（ページ内アンカー）へのリンクを置く

## ページ構成

単一ページ（`(app)/help/page.tsx`）に機能ごとのセクションをアンカー区切りで並べる構成。トピック一覧+個別ページに分ける構成は今回の規模（カテゴリ管理1項目のみ）には過剰なため採用しない。将来他機能のヘルプが必要になった時点で同じページにセクションを追加する。

`(app)/layout.tsx`配下に配置し、ヘッダー・下部固定ナビゲーション等の共通レイアウトに乗せる（ヘルプページ自体はナビの5項目には含まれない）。

## カテゴリ管理セクション（`#categories`）

現時点で唯一実装するセクション。以下2点を説明する。

### 親カテゴリ機能

[親カテゴリ機能](./categories.md#親カテゴリグラフ上のグルーピング)の効果（子カテゴリの金額を親カテゴリに合算してグラフ上1枠にまとめる）を、テキストに加えて簡易図解で説明する。

- 図解は「電気代」「ガス代」「水道代」の3つの円形カテゴリアイコン → 矢印 → 「光熱費」の円グラフ1枠、という構成
- 実装はStitchによる画像生成ではなく、Tailwind+SVGでコード側に直接組む（[カテゴリ一覧のアイコンバッジ](../../design/categories/list.md#採用した方向性)と同じビジュアル言語を再利用できる、画像アセット管理・Stitch生成コストが不要という理由）。実装を進める中で見た目の都合上画像の方が適切と判断した場合は切り替えてよい

### ピン留め機能

[ピン留め機能](./categories.md#カテゴリの固定表示ピン留め)の効果（ピン留めしたカテゴリは金額のランキングに関わらず必ずホームの円グラフに表示される）を説明する。図解は必須ではなく、テキストのみで足りると判断する（ピン自体のメタファーが直感的なため、親カテゴリほど図解の必要性は高くない）。
